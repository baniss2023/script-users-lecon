Gestion des utilisateurs et des groupes en ligne de commande sous Windows

Cette leçon présente les notions d’utilisateur et de groupe sous Windows, puis montre comment créer, afficher, modifier, désactiver et supprimer des comptes et des groupes à l’aide des commandes net user et net localgroup. Elle se termine par des exemples d’automatisation utiles pour l’administration courante.

Table des matières

  • Comprendre les notions de base
  • Créer et gérer les groupes locaux
  • Créer et gérer les utilisateurs
  • Automatiser et récapituler les opérations essentielles
  • QCM final de validation

Objectifs de la leçon

  • Créer et gérer des groupes locaux : création, suppression et affichage.
  • Créer des utilisateurs puis les associer aux groupes existants.
  • Afficher la liste des utilisateurs d’un système Windows avec différents niveaux de détails.
  • Supprimer des utilisateurs et les retirer de groupes.
  • Automatiser certaines tâches courantes liées à la gestion des utilisateurs.

Comprendre les notions de base

Notion d’utilisateur

Dans un système Windows, un utilisateur est une entité, une personne ou un service, qui possède un compte lui permettant d’accéder à l’ordinateur avec un ensemble de permissions déterminées. Un compte utilisateur peut comporter un nom, un mot de passe et des autorisations spécifiques.

  • Des utilisateurs locaux, définis sur une seule machine.
  • Des utilisateurs standards, qui disposent de droits limités.
  • Des administrateurs, qui disposent de droits élevés.
Idée clé : un compte utilisateur permet d’identifier précisément qui accède à la machine et avec quels droits.

Notion de groupes

Un groupe est un ensemble d’utilisateurs qui partagent les mêmes droits ou les mêmes fonctions. Grâce aux groupes, on peut gérer les autorisations de manière collective, ce qui évite de configurer les droits utilisateur par utilisateur.

Le document distingue trois grandes catégories de groupes sous Windows :

  • Les groupes locaux, définis sur une machine spécifique et valables uniquement sur cette machine.
  • Les groupes prédéfinis, comme Administrateurs, Utilisateurs ou Invités.
  • Les groupes personnalisés, créés par l’administrateur pour organiser les utilisateurs selon des rôles, des services ou des projets.

Par exemple, on peut accorder un accès à un dossier partagé à tous les membres d’un groupe comme Comptabilité sans ajouter chaque utilisateur individuellement.

Pourquoi c’est utile : les groupes simplifient la gestion des permissions et rendent l’administration d’un poste plus rapide et plus cohérente.

Créer et gérer les groupes locaux

Créer un groupe local

Pour créer un groupe local personnalisé, on utilise la commande net localgroup suivie du nom du groupe et de l’option /add.

Syntaxe générale

net localgroup NomGroupe /add

Exemple simple

net localgroup Comptables /add

Exemples successifs

net localgroup GroupeA /add
net localgroup GroupeB /add
net localgroup GroupeC /add
Administrateur : Windows PowerShell
Création de plusieurs groupes avec une boucle PowerShell
1$groupes = @("GroupeA", "GroupeB", "GroupeC")
2foreach ($groupe in $groupes) {
3 net localgroup $groupe /add
4 Write-Output "Groupe $groupe créé."
5}

Supprimer un groupe local

Pour supprimer un groupe, on utilise la même commande avec l’option /delete.

Syntaxe générale

net localgroup NomGroupe /delete

Exemple

net localgroup Secretaires /delete
Prudence : avant de supprimer un groupe, il est recommandé de vérifier qu’il n’est plus utilisé pour accorder des droits importants.

Afficher tous les groupes locaux

Pour afficher la liste complète des groupes définis localement sur le système, la commande de base est la suivante :

Commande à utiliser

net localgroup

Cette commande permet de vérifier quels groupes sont déjà présents sur la machine avant d’en créer de nouveaux ou d’en supprimer.

Créer et gérer les utilisateurs

Créer un utilisateur avec un mot de passe

Pour créer un utilisateur en ligne de commande, on utilise net user en indiquant le nom du compte, le mot de passe puis l’option /add.

Syntaxe générale

net user NomUtilisateur MotDePasse /add

Exemple

net user Etudiant1 P@ssw0rd! /add

Créer plusieurs utilisateurs à partir d’une liste

Le document présente ensuite un script PowerShell qui crée plusieurs utilisateurs automatiquement. Le script s’appuie sur une liste de noms, génère un mot de passe puis crée les comptes un par un.

Administrateur : Windows PowerShell
Création automatique de plusieurs utilisateurs
1$users = @("User10", "User11", "User12")
2function Generer-MotDePasse {
3 $lettres = "abcdefghijklmnopqrstuvwxyzABCDEFGHIJKLMNOPQRSTUVWXYZ".ToCharArray()
4 $chiffres = "0123456789".ToCharArray()
5 $speciaux = "!@#$%*?".ToCharArray()
6 $mot = -join (1..6 | ForEach-Object { Get-Random -InputObject $lettres })
7 $mot += Get-Random -InputObject $chiffres
8 $mot += Get-Random -InputObject $speciaux
9 return $mot
10}
11Write-Output "==========================================="
12Write-Output "Création des utilisateurs avec mot de passe"
13foreach ($u in $users) {
14 $pw = Generer-MotDePasse
15 net user $u $pw /add
16 Write-Output "Utilisateur : $u Mot de passe : $pw"
17}
Observation : la sortie affiche le nom de chaque utilisateur créé ainsi que le mot de passe attribué pendant l’exécution du script.

Ajouter des utilisateurs à des groupes

L’ajout d’un utilisateur à un groupe se fait à l’aide de la commande suivante :

Syntaxe générale

net localgroup NomGroupe NomUtilisateur /add

Exemples

net localgroup GroupeA User1 /add
net localgroup GroupeA User2 /add
net localgroup GroupeB User3 /add
net localgroup GroupeB User4 /add
net localgroup GroupeB User5 /add

Modifier les informations d’un utilisateur

Pour modifier un utilisateur, par exemple pour changer son mot de passe, on utilise la commande suivante :

Syntaxe générale

net user NomUtilisateur NouveauMotDePasse

Exemple

net user User1 NewP@ss123

Cette vérification est utile pour confirmer la présence du compte avant de le modifier.

Afficher la liste des utilisateurs du système

La commande de base pour afficher tous les utilisateurs d’un système Windows est :

Commande à utiliser

net user
  • Cette commande liste tous les utilisateurs locaux définis sur l’ordinateur.
  • Elle est utile pour vérifier la présence d’un compte avant de le modifier.

Afficher les détails d’un utilisateur

Pour obtenir plus d’informations sur un utilisateur spécifique, on utilise la commande suivante :

Commande à utiliser

net user NomUtilisateur
  • Il faut remplacer NomUtilisateur par le nom réel du compte.
  • La commande affiche notamment le groupe auquel appartient l’utilisateur, la date de dernière connexion et plusieurs informations liées au compte.

Afficher les utilisateurs d’un groupe spécifique

Pour voir quels utilisateurs appartiennent à un groupe spécifique, on indique simplement le nom du groupe après net localgroup.

Syntaxe générale

net localgroup NomGroupe

Exemple

net localgroup Administrateurs

Supprimer un utilisateur d’un groupe et supprimer un utilisateur du système

Pour retirer un utilisateur d’un groupe, on utilise l’option /delete sur la commande net localgroup. Pour supprimer totalement un utilisateur du système, on utilise l’option /delete avec net user.

Retirer un utilisateur d’un groupe

net localgroup NomGroupe NomUtilisateur /delete

Exemples :
net localgroup GroupeA User1 /delete
net localgroup GroupeB User3 /delete

Supprimer un utilisateur du système

net user NomUtilisateur /delete

Exemple :
net user User1 /delete
Différence importante : retirer un utilisateur d’un groupe ne supprime pas son compte ; cela supprime seulement son appartenance à ce groupe.

Désactiver un utilisateur sans le supprimer

Plutôt que de supprimer un utilisateur, on peut le désactiver temporairement. Cette approche permet de conserver le compte sans le rendre actif.

Pour désactiver

net user NomUtilisateur /active:no

Exemple :
net user User1 /active:no

Pour réactiver

net user User1 /active:yes

Automatiser et récapituler les opérations essentielles

Exemples d’automatisation des tâches

La dernière partie du document rappelle que plusieurs opérations peuvent être automatisées dans le contexte de la gestion des utilisateurs et des groupes.

  • Créer des utilisateurs automatiquement à partir d’une liste.
  • Attribuer automatiquement un mot de passe sécurisé à l’aide d’un générateur aléatoire.
  • Créer un script de suppression automatique des utilisateurs inactifs.
  • Envoyer un rapport périodique des comptes créés et supprimés.
  • Programmer la désactivation des comptes inactifs après un certain temps.
  • Créer des groupes puis y ajouter des utilisateurs automatiquement.
  • Vérifier régulièrement les droits des utilisateurs et signaler des modifications suspectes.
  • Réinitialiser automatiquement les mots de passe des comptes expirés par script PowerShell.
À retenir : l’automatisation permet de gagner du temps, de réduire les erreurs répétitives et de standardiser les tâches d’administration.
